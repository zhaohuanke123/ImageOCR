# OCR Pipeline Frontend

React frontend for the OCR Pipeline review tool. This application provides an interactive interface for reviewing, editing, and exporting OCR results from mind map images.

## Features

- **Review Interface**: Visualize OCR nodes, edges, and issues on the original image
- **Node Editing**: Edit, delete, merge, and split nodes
- **Edge Editing**: Add, modify, and delete parent-child relationships
- **Baseline Editor**: Create and edit baseline annotation datasets for regression testing
- **Regression Reports**: View and analyze test results with visual comparisons
- **Export Options**: Export as JSON or Markdown outline

## Prerequisites

- Node.js 18+ and npm
- Backend OCR pipeline (Python)

## Installation

```bash
# Navigate to frontend directory
cd frontend

# Install dependencies
npm install
```

## Development Mode

Start the development server:

```bash
npm run dev
```

The application will be available at `http://localhost:5173`.

### Development Mode Requirements

In development mode, the frontend expects:
1. `frontend/public/graph.json` - OCR graph data
2. `frontend/public/GameEngine.jpg` - Image file (or update the path in App.tsx)

To generate this data:
```bash
# From the project root
uv run ocr-pipeline graph --input artifacts/ocr_merged.json

# Copy generated files to frontend/public
cp artifacts/graph.json frontend/public/
```

## Production Build

Build for production:

```bash
npm run build
```

The output will be in `frontend/dist/`.

## Integration with OCR Pipeline

The frontend is designed to be embedded in the HTML generated by `ocr-pipeline review-html`. The backend CLI:

1. Builds the React app
2. Injects the review data as JSON
3. Serves as a single HTML file

### Using with Backend

```bash
# Generate review HTML with embedded frontend
uv run ocr-pipeline review-html --image GameEngine.jpg --graph artifacts/graph.json

# Output: artifacts/review/index.html
```

## Pages and Routing

The application uses hash-based routing:

| Route | Description |
|-------|-------------|
| `#/review` | Main review page (default) |
| `#/baseline-editor?dataset=<name>&region=<id>` | Baseline editor |
| `#/regression` | Regression test report |

## User Guide

### Review Page

The main interface for reviewing OCR results.

#### Sidebar Controls

- **Search**: Filter nodes by text content
- **Show OCR blocks**: Toggle raw OCR detection visibility
- **Show nodes**: Toggle merged node visibility
- **Show edges**: Toggle parent-child edge visibility
- **Only roots**: Filter to show only root nodes (no parent)
- **Only low confidence**: Filter to show only low confidence nodes

#### Issue Panel

Lists detected issues:
- **Orphan root**: Root nodes that may not be actual roots
- **Low confidence**: Nodes with confidence below threshold
- **Oversized node**: Nodes with unusually large bounding boxes
- **Weak edge**: Parent-child relationships with low confidence scores

#### Node Panel

Shows the node tree structure:
- Click to select and highlight on image
- Double-click to edit node properties
- Use checkboxes to select multiple nodes for merging

#### Image Viewer

- **Pan**: Drag with mouse or use arrow keys
- **Zoom**: Scroll wheel or +/- buttons
- **Brightness/Contrast**: Adjust image for better visibility
- **Tool modes**: Select, Pan, Box Select

### Node Editing

#### Edit Node
- Double-click a node in the panel or on the image
- Modify text, bounding box, or confidence
- Changes are tracked for save/export

#### Delete Node
- Select node and press Delete key
- Or use the delete button in the edit modal
- Children are re-parented to the deleted node's parent

#### Merge Nodes
1. Select multiple nodes using checkboxes
2. Click "Merge" button
3. A new merged node is created with combined text

#### Split Node
- In the edit modal, split multi-line nodes into separate nodes
- Each line becomes a new node

### Edge Editing

#### Add Edge
1. Click the edge tool button
2. Select parent and child nodes
3. The system prevents cycles and self-loops

#### Delete Edge
- Click on an edge in the image or panel
- Use the delete option in the edit modal

### Saving Changes

- **Ctrl+S**: Open save dialog
- Saves modified `graph.json` and `change_log_<date>.json`
- Change log records all operations for audit trail

### Export Options

- **JSON**: Export nodes and edges as `graph.json`
- **Markdown**: Export hierarchical outline as `mindmap_outline.md`

### Baseline Editor

Create and edit baseline annotations for regression testing.

#### Loading Data
- Open a region JSON file from `baseline/<dataset>/region_XX.json`
- Or use URL parameters: `#/baseline-editor?dataset=GameEngine&region=region_01`

#### Editing
- Add/edit/delete blocks, nodes, and edges
- Draw new annotations on the image
- Save changes to the region JSON file

### Regression Report

View regression test results.

#### Loading Results
- Open `reports/summary.json` from a regression test
- Or run a new test from the interface

#### Analysis
- Overview tab: Summary metrics and failed regions
- Regions tab: Detailed per-region results
- History tab: Compare results across test runs

- Safari: File System Access API not supported (falls back to download)
- Mobile: Touch gestures partially supported

## Performance Considerations

- Large images (>10,000px) may require significant memory
- Recommended: Maximum 500 nodes for smooth interaction
- Use filters to reduce visible elements

## Troubleshooting

### "No data loaded" error
- Ensure `graph.json` exists in `public/` directory (dev mode)
- Or use the embedded data from backend generation

### Image not displaying
- Check image path in the data matches an accessible file
- Dev mode: Copy image to `frontend/public/`

### Slow performance
- Reduce visible elements using filters
- Consider splitting large images into regions
