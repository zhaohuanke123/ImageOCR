## 一、功能列表与用户故事

| 核心显示 | 节点叠加显示 | P0 | 在图片上叠加显示节点边界框和文本 |
| 核心显示 | 边连线显示 | P0 | 显示节点间的父子关系连线 |
| 图像操作 | 缩放平移 | P0 | 支持大图的缩放和平移浏览 |
| 图像操作 | 图像调整 | P0 | 亮度、对比度调整，辅助 OCR 识别 |
| 补录功能 | 框选 OCR | P0 | 框选遗漏区域，触发 OCR 补录 |

| 问题管理 | 问题列表 | P1 | 显示检测到的问题节点 |
| 问题管理 | 问题导航 | P1 | 快速定位问题节点 |

| 导出功能 | 结果导出 | P0 | 导出 JSON、Markdown 等格式 |

### 1.2 用户故事

#### US-001: 主视图展示
> **作为** 用户
> **我希望** 看到思维导图图片和 OCR 识别结果叠加显示
> **以便于** 直观地检查识别的准确性和完整性

**验收标准**:
- 页面加载后自动显示图片和标注
- 标注框清晰可见，不遮挡文本
- 支持标注框的显示/隐藏切换

#### US-002: 缩放平移
> **作为** 用户
> **我希望** 能够自由缩放和平移查看大图
> **以便于** 检查图片的各个细节区域

**验收标准**:
- 滚轮缩放，以鼠标位置为中心
- 空格+拖拽或中键拖拽平移
- 显示当前缩放比例
- 支持适应窗口/实际大小快捷操作

#### US-003: 图像调整
> **作为** 用户
> **我希望** 调整图片的亮度和对比度
> **以便于** 在光照不佳的区域也能清晰查看文本

**验收标准**:
- 亮度/对比度可通过按钮或滑块调整
- 调整效果实时预览
- 支持一键重置到原始状态
- 调整范围: -100 到 +100

#### US-004: 框选 OCR 补录
> **作为** 用户
> **我希望** 框选图片上遗漏的区域进行 OCR 补录
> **以便于** 补充 OCR 遗漏的文本内容

**验收标准**:
- 鼠标拖拽绘制矩形选框
- 松开鼠标后自动触发 OCR
- OCR 过程显示加载状态
- 识别结果自动添加到节点列表
- 框选区域最小 20x20 像素

#### US-005: 问题列表
> **作为** 用户
> **我希望** 看到所有问题节点的列表
> **以便于** 快速定位需要人工复核的问题

**验收标准**:
- 显示问题类型图标和描述
- 点击问题项跳转到对应节点
- 显示问题统计数量
- 支持按问题类型筛选

#### US-006: 节点列表
> **作为** 用户
> **我希望** 看到所有识别节点的列表
> **以便于** 浏览和搜索特定节点

**验收标准**:
- 显示节点文本预览和置信度
- 支持文本搜索过滤
- 点击节点在主视图高亮显示
- 显示节点总数

#### US-007: 结果导出
> **作为** 用户
> **我希望** 导出最终的识别结果
> **以便于** 在其他工具中使用结构化数据

**验收标准**:
- 支持 JSON、Markdown 格式导出
- 导出时包含节点和边信息
- 显示导出成功提示和文件路径

## 二、功能 UX/UI 考量

### 2.1 主视图展示

#### 核心体验目标
- **视觉重心**: 主视图占据页面主体空间，用户注意力聚焦于图像内容
- **即时反馈**: 任何操作立即响应，无感知延迟
- **信息分层**: 图片 → 标注框 → 文本标签，层次清晰

#### UI 设计
```
┌──────────────────────────────────────────────────────────────────┐
│  工具栏: [亮度 -] [亮度 +] [对比度 -] [对比度 +] [重置] [框选OCR] │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│   ┌─────────────────────────────────────────────────────────┐    │
│   │                                                          │    │
│   │              思维导图图片主视图                           │    │
│   │              （可缩放、可框选）                           │    │
│   │                                                          │    │
│   │    ┌─────────┐                                          │    │
│   │    │ 渲染管线 │  ← 节点标注框 + 文本标签                  │    │
│   │    └─────────┘                                          │    │
│   │         │                                               │    │
│   │         ↓  ← 边连线                                     │    │
│   │    ┌─────────┐                                          │    │
│   │    │ 光照计算 │                                         │    │
│   │    └─────────┘                                          │    │
│   │                                                          │    │
│   └─────────────────────────────────────────────────────────┘    │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

#### 状态描述

| 状态 | 视觉表现 | 用户操作 |
|------|---------|---------|
| 加载中 | 显示骨架屏或加载动画 | 等待 |
| 正常 | 图片 + 标注叠加显示 | 缩放/平移/框选/调整 |
| 框选模式 | 鼠标变为十字光标 | 拖拽绘制选框 |
| OCR 进行中 | 选框区域显示 loading 动画 | 等待 |

#### 动画设计
- **缩放**: CSS transform，duration 200ms，ease-out
- **平移**: CSS transform，duration 100ms
- **标注框出现**: CSS opacity 0→1，duration 150ms
- **OCR loading**: CSS 旋转动画，duration 1s，linear infinite

---

### 2.2 缩放平移

#### 核心体验目标
- **流畅性**: 缩放平移过程平滑，无明显卡顿
- **精准性**: 缩放以鼠标位置为中心，符合用户预期
- **可控性**: 提供多种操作方式，满足不同习惯

#### 交互设计

| 操作 | 方式 | 优先级 |
|------|------|--------|
| 缩放 | 鼠标滚轮 | 主要 |
| 缩放 | 工具栏 +/- 按钮 | 备选 |
| 缩放 | Ctrl + 滚轮 | 备选 |
| 平移 | 空格 + 左键拖拽 | 主要 |
| 平移 | 鼠标中键拖拽 | 备选 |
| 平移 | 方向键 | 微调 |
| 适应窗口 | 双击/按钮 | 快捷 |
| 实际大小 | 按钮 | 快捷 |

#### 状态显示
- 缩放比例指示器: 右下角显示 "100%" 或 "50%"
- 最小缩放: 10%
- 最大缩放: 500%

#### 边界处理
- 平移到边界时: 阻尼效果，图片不完全移出视口
- 缩放到极限时: 无法继续缩放，无操作反馈

---

### 2.3 图像调整

#### 核心体验目标
- **实时性**: 调整效果即时预览，无需等待
- **可逆性**: 随时可以重置到原始状态
- **适度性**: 调整范围合理，避免过度处理

#### UI 设计
```
┌──────────────────────────────────────────────────┐
│  [亮度 -] [亮度 +]  [对比度 -] [对比度 +]  [重置] │
│                                                  │
│   当前: 亮度 +10  对比度 +5                      │
└──────────────────────────────────────────────────┘
```

#### 调整参数

| 参数 | 范围 | 步进 | 默认值 |
|------|------|------|--------|
| 亮度 | -100 ~ +100 | 10 | 0 |
| 对比度 | -100 ~ +100 | 10 | 0 |

#### 技术实现
```javascript
// CSS Filter 实现实时预览
imageElement.style.filter = `brightness(${100 + brightness}%) contrast(${100 + contrast}%)`;
```

#### 状态管理
- 调整值保存在内存中，不持久化
- 页面刷新后恢复默认值
- 框选 OCR 时使用当前调整后的图像

---

### 2.4 框选 OCR 补录

#### 核心体验目标
- **直观性**: 框选操作简单明了
- **反馈性**: OCR 过程有明确的状态反馈
- **高效性**: 框选后自动触发，无需额外确认

#### 交互流程
```
1. 点击 [框选OCR] 按钮 → 进入框选模式
2. 鼠标变为十字光标
3. 按住左键拖拽 → 绘制矩形选框
4. 松开左键 → 触发 OCR 请求
5. 显示 loading 动画
6. OCR 完成 → 显示识别结果
7. 自动退出框选模式
```

#### UI 设计 - 框选过程
```
┌─────────────────────────────────────────────────┐
│                                                 │
│     ┌───────────────────┐                      │
│     │ █████████████████ │ ← 选框区域           │
│     │ █████████████████ │   (半透明蒙层)       │
│     │ █████████████████ │                      │
│     └───────────────────┘                      │
│           ↑ 选框边界线                          │
└─────────────────────────────────────────────────┘
```

#### 状态机设计
```
                    ┌─────────────┐
                    │   默认状态   │
                    └──────┬──────┘
                           │ 点击框选按钮
                           ▼
                    ┌─────────────┐
                    │  框选模式    │ ←──┐
                    └──────┬──────┘    │
                           │ 绘制选框   │ 按 ESC 取消
                           ▼           │
                    ┌─────────────┐    │
                    │  OCR 进行中  │────┘
                    └──────┬──────┘
                           │ OCR 完成
                           ▼
                    ┌─────────────┐
                    │  结果展示    │
                    └──────┬──────┘
                           │ 确认
                           ▼
                    ┌─────────────┐
                    │   默认状态   │
                    └─────────────┘
```

#### OCR 结果展示
- 成功: 在选框位置显示新的标注框，弹出提示 "识别到 X 个文本块"
- 无结果: 显示提示 "该区域未识别到文字，请尝试调整对比度"
- 失败: 显示错误提示 "OCR 服务连接失败，请检查后端服务"

#### 防抖处理
- 连续快速框选: 300ms 内的重复请求被忽略
- OCR 进行中: 禁用新的框选操作

### 2.5 问题列表

#### 核心体验目标
- **可见性**: 问题一目了然，不需要主动查找
- **可操作性**: 点击即可定位，快速处理
- **分类清晰**: 不同问题类型有明确的视觉区分

#### UI 设计
```
┌──────────────────────────────────────────────────────────────────┐
│  问题列表 (3)                                          [全部展开] │
├──────────────────────────────────────────────────────────────────┤
│  ├─ ⚠️ 孤立节点: "渲染管线" (无父节点)           [定位]          │
│  ├─ ⚠️ 低置信度: "光照计算" (0.42)               [定位]          │
│  └─ ⚠️ 弱连接: "阴影" → "月影" (score: 0.31)     [定位]          │
└──────────────────────────────────────────────────────────────────┘
```

#### 问题类型与视觉标记

| 问题类型 | 图标 | 颜色 | 说明 |
|---------|------|------|------|
| 孤立节点 | ⚠️ | 红色边框 | 没有父节点的节点 |
| 低置信度 | ⚠️ | 黄色边框 | 置信度低于阈值 |
| 弱连接 | ⚠️ | 虚线边框 | 边分数低于阈值 |
| 超大节点 | ⚠️ | 橙色边框 | 节点尺寸超过阈值 |

#### 交互设计
- 点击问题项: 主视图平移到对应节点，节点高亮闪烁
- 双击问题项: 打开节点详情编辑
- [全部展开]: 展开所有问题的详细信息

### 2.7 结果导出

#### 核心体验目标
- **确定性**: 明确告知导出结果位置
- **完整性**: 导出所有必要信息

| 格式 | 内容 | 用途 |

- 点击 [导出结果] 按钮

## 四、交互流程图（文字描述）

### 4.1 页面加载流程

```
用户打开 HTML 文件
    │
    ▼
浏览器解析 HTML 结构
    │
    ▼
加载图片资源
    │
    ├─ 成功 ─→ 显示图片
    │           │
    │           ▼
    │       加载 OCR 结果 JSON
    │           │
    │           ├─ 成功 ─→ 渲染标注框和边连线
```

## 五、边缘案例处理

### 5.1 图片加载失败

| 场景 | 处理方式 |
|------|---------|
| 文件路径错误 | 显示 "图片文件不存在，请检查路径" |
| 文件格式不支持 | 显示 "不支持的图片格式，支持 jpg/png/bmp" |
| 文件损坏 | 显示 "图片文件损坏或无法读取" |
| 网络超时 | 显示 "图片加载超时，请刷新页面重试" |

### 5.2 OCR 处理边缘案例

| 场景 | 处理方式 |
|------|---------|
| 框选区域过小 (< 20x20) | 提示 "框选区域过小，请扩大选择范围" |
| OCR 无结果 | 提示 "该区域未识别到文字，请尝试调整对比度后重试" |
| 后端服务未启动 | 提示 "OCR 服务未启动，请运行 ocr-pipeline serve-editor-ocr" |
| 请求超时 (> 30s) | 提示 "OCR 请求超时，请重试" |
| 连续快速框选 | 300ms 防抖，忽略重复请求 |

### 5.3 图像调整边缘案例

| 场景 | 处理方式 |
|------|---------|
| 亮度/对比度到达边界值 | 按钮禁用，无法继续调整 |
| 调整后图片不清晰 | 提示建议 "对比度建议调整范围: -50 ~ +50" |
| 重置操作 | 一键恢复所有参数到默认值 |

### 5.4 缩放平移边缘案例

| 场景 | 处理方式 |
|------|---------|
| 缩放到最小值 (10%) | 无法继续缩小，无操作反馈 |
| 缩放到最大值 (500%) | 无法继续放大，无操作反馈 |
| 平移到边界 | 阻尼效果，图片不完全移出视口 |
| 大图内存不足 | 提示 "图片过大，建议关闭其他标签页" |

### 5.5 数据边缘案例

| 场景 | 处理方式 |
|------|---------|
| 无 OCR 结果数据 | 显示空状态 "暂无识别结果，请先运行 OCR" |
| 无问题节点 | 问题列表显示 "无待处理问题" |
| 节点列表为空 | 显示 "暂无节点数据" |
| 搜索无结果 | 显示 "未找到匹配节点" |

## 六、UX 原则总结

### 6.1 简洁性原则

1. **界面简洁**: 布局与 PRD 线框图一致，顶部工具栏 + 主视图 + 底部列表，无冗余元素
2. **操作简洁**: 框选 OCR 一气呵成，无需多步确认
3. **文案简洁**: 遵循 PRD 文案规范，信息密度高，无废话

### 6.2 效率原则

1. **即时反馈**: 任何操作立即响应，无感知延迟
2. **快捷键支持**: 提供常用操作的键盘快捷方式
3. **自动处理**: OCR 补录后自动更新节点列表和问题列表

### 6.3 可访问性原则

1. **视觉可访问**:
   - 足够的颜色对比度
   - 问题标记不依赖颜色区分（使用图标+颜色）
   - 字体大小适中

2. **操作可访问**:
   - 提供多种交互方式（鼠标、键盘）
   - 操作状态有明确的视觉反馈
   - 错误提示清晰，包含解决建议

3. **认知可访问**:
   - 界面布局符合用户心智模型
   - 功能分区明确
   - 操作流程可预测

### 6.4 容错性原则

1. **操作可逆**: 图像调整可重置（注: 不支持撤销/重做）
2. **错误恢复**: 错误提示包含解决方案
3. **防误操作**: 框选区域过小时提示并允许重试

### 6.5 一致性原则

1. **视觉一致**: 使用统一的设计语言和图标风格
2. **交互一致**: 相同类型操作使用相同的交互方式
3. **反馈一致**: 同类操作使用相同的反馈形式

## 七、技术实现建议

### 7.1 前端技术选型

| 功能模块 | 推荐技术 | 理由 |
|---------|---------|------|
| 画布渲染 | Fabric.js / Konva.js | 支持对象选择、变换、事件 |
| 大图处理 | OpenSeadragon / Leaflet | 分块加载，支持超大图 |
| 状态管理 | Zustand / Jotai | 轻量，适合复杂状态 |
| 快捷键 | hotkeys-js | 成熟的快捷键库 |

### 7.2 性能优化建议

1. **图片渲染**: 使用 CSS transform 而非修改图片尺寸
2. **标注叠加**: 使用 SVG 或 Canvas，避免大量 DOM 元素
3. **图像调整**: 使用 CSS Filter，GPU 加速
4. **列表渲染**: 虚拟滚动，仅渲染可见项

### 7.3 兼容性要求

- 浏览器: Chrome 90+, Edge 90+
- 操作系统: Windows 10/11
- 图片格式: jpg, png, bmp
